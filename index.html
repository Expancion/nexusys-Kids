Quick start

Up and running in minutes

1

Clone and configure

Copy the example env file and fill in your secrets.

# clone the repo
git clone https://github.com/your-username/nexusys-junior.git
cd nexusys-junior

# create .env from template
cp .env.example .env

Set at minimum:

SECRET_KEY=your-random-secret
POSTGRES_PASSWORD=your-db-password
DATABASE_URL=postgresql://nexusys:your-db-password@db:5432/nexusys
NEXUSYS_PUBLIC_URL=https://your-domain.local   # used in NixOS config download
YOUTUBE_API_KEY=optional-for-playlist-import
2

Start

docker compose up -d
3

Run the setup wizard

Open your browser — you'll be redirected to /setup automatically.

Choose your language (CZ / EN), set the admin password, done. You land directly in the admin panel.

4

Access

URLDescription
http://localhostKiosk home screen
http://localhost/adminParent admin panel
http://localhost/kiosk/1Child kiosk (replace 1 with child ID)
http://localhost/admin/helpBilingual user guide
http://localhost/api/REST API documentation
💻

NixOS kiosk: After adding a child's device in the admin panel, click ↓ NixOS to download a pre-filled .nix config file. Copy to /etc/nixos/configuration.nix, place your Caddy CA cert, run nixos-rebuild switch. The laptop boots into a locked kiosk — the agent polls the server every 15 seconds.

Security

Secure by default

🔐

Hashed passwords

Admin password is stored as a scrypt hash (werkzeug 3.x) in the database. Never in plaintext or env files.

🌐

Automatic HTTPS

Caddy handles TLS automatically — including internal CA for LAN-only deployments without a public domain.

❄️

Locked kiosk OS

The kid NixOS user has no sudo. KDE wallet, screen lock, and power keys are all disabled.
