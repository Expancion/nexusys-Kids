GET /api/health

Liveness probe. Returns 200 when the app is up.

RESPONSE 200

{"status": "ok", "service": "nexus-junior"}

NIXOS USAGE

curl -fs https://nexusys-kid.home.prod/api/health | jq .status
GET /api/device-policy

Main gate for the NixOS kiosk agent. Checks whether a device is allowed to run and returns child data needed to open the kiosk. Also incorporates time-based schedule locks — if a ChildSchedule lock is active the response is allowed: false even if the device itself is enabled.

QUERY PARAMETERS
deviceId required The unique device key registered in Admin → Child → Devices. E.g. my-device-01

RESPONSE 200 — allowed

{
  "allowed": true,
  "childId": 1,
  "childName": "child-name",
  "points": 42,
  "videoCost": 10,
  "maxVideos": 4,
  "dailyLimit": 5,
  "message": "Máš 42 bodů (4 videí).",
  "kioskUrl": "/kiosk/1"
}

RESPONSE 200 — blocked by admin

{
  "allowed": false,
  "childName": "child-name",
  "message": "Kiosek je teď nedostupný."
}

RESPONSE 200 — blocked by time schedule schedule_locked

{
  "allowed": false,
  "childName": "child-name",
  "message": "Kiosek je zamčený. Jdi spát! 🌙",
  "schedule_locked": true
}

RESPONSE 404 — unknown device

{"allowed": false, "message": "Zařízení nebylo nalezeno."}

DEVICE_KEY="my-device-01"
BASE="https://nexusys-kid.home.prod"

policy=$(curl -sf "$BASE/api/device-policy?deviceId=$DEVICE_KEY")
allowed=$(echo "$policy" | jq -r '.allowed')

if [ "$allowed" = "true" ]; then
  kiosk_url=$(echo "$policy" | jq -r '.kioskUrl')
  # launch browser → $BASE$kiosk_url
  chromium --kiosk "$BASE$kiosk_url"
else
  msg=$(echo "$policy" | jq -r '.message')
  # show lock screen with $msg
fi
POST /api/kiosk/watch

Records that a child started watching a video and deducts the point cost. Blocked server-side if a schedule lock is active — cannot be bypassed from the client.

REQUEST BODY (JSON)
child_id required integer — from device-policy response
video_id optional integer — used for time-based price rules; if omitted uses child default cost
device_key optional string — stored in WatchSession for audit

RESPONSE 200 — ok

{"ok": true, "new_balance": 32}

RESPONSE 200 — not enough points

{
  "ok": false,
  "reason": "not_enough_points",
  "message": "Nemáš dost bodů. Vydělej si je splněním úkolů!",
  "balance": 3
}

RESPONSE 403 — schedule locked schedule_locked

{
  "ok": false,
  "reason": "schedule_locked",
  "message": "Kiosek je zamčený. Jdi spát! 🌙"
}
POST /api/kiosk/chore/complete

Marks a daily chore as done for today and awards points. Idempotent within the same day — calling it twice returns already_done. Not blocked by schedule locks (physical chores can be completed at any time).

REQUEST BODY (JSON)
child_id required integer
chore_id required integer — from kiosk page chores data

RESPONSE 200 — ok

{
  "ok": true,
  "awarded": 5,
  "new_balance": 47,
  "chore_name": "Uklidit pokoj"
}

RESPONSE 200 — already done today

{
  "ok": false,
  "reason": "already_done",
  "message": "Úkol 'Uklidit pokoj' jsi dnes už splnil! 🎉"
}

RESPONSE 403 — wrong child

{"ok": false, "reason": "not_your_chore"}

NixOS Kiosk Agent Skeleton

#!/usr/bin/env bash
# /etc/nexus-junior/kiosk-agent.sh
# Runs as a systemd service on the NixOS kiosk device.
# Polls device-policy every 60 s; relaunches browser on change.

set -euo pipefail

BASE="https://nexusys-kid.home.prod"
DEVICE_KEY="${NEXUS_DEVICE_KEY:-{{ first_device_key }}}"
POLL_INTERVAL=60
LOCK_MSG_FILE="/run/nexus-lock-msg"
BROWSER_PID=""

log() { echo "[nexus-agent] $*"; }

kill_browser() {
  [ -n "$BROWSER_PID" ] && kill "$BROWSER_PID" 2>/dev/null || true
  BROWSER_PID=""
}

launch_kiosk() {
  local url="$1"
  kill_browser
  # adjust binary for your DE (chromium / firefox / cage + surf …)
  chromium \
    --kiosk \
    --no-first-run \
    --disable-infobars \
    --noerrdialogs \
    "$url" &
  BROWSER_PID=$!
  log "Launched kiosk → $url (pid $BROWSER_PID)"
}

show_lock_screen() {
  local msg="$1"
  echo "$msg" > "$LOCK_MSG_FILE"
  kill_browser
  # swap in your lock-screen binary / cage app here
  log "Lock screen: $msg"
}

check_policy() {
  local policy allowed kiosk_url msg

  policy=$(curl -sf --max-time 5 \
    "$BASE/api/device-policy?deviceId=$DEVICE_KEY") || {
    log "Policy check failed (network?), keeping current state."
    return
  }

  allowed=$(echo "$policy" | jq -r '.allowed')

  if [ "$allowed" = "true" ]; then
    kiosk_url=$(echo "$policy" | jq -r '.kioskUrl')
    # only relaunch if not already showing the right URL
    launch_kiosk "$BASE$kiosk_url"
  else
    msg=$(echo "$policy" | jq -r '.message')
    show_lock_screen "$msg"
  fi
}

# ── main loop ────────────────────────────────────────────────────
log "Starting. device=$DEVICE_KEY  base=$BASE"
check_policy
while sleep "$POLL_INTERVAL"; do
  check_policy
done

systemd unit — add Environment=NEXUS_DEVICE_KEY=my-device-01 in the [Service] block.

TLS cert — trust ./certs/caddy-root-ca.crt (exported by Caddy on build). Copy to /etc/ssl/certs/ and run update-ca-certificates.

Schedule locking — handled server-side. The agent doesn't need to know about lock windows; device-policy returns allowed: false automatically when a time lock is active.
